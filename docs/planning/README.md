# Planning Docs Index

Use this folder as the source of truth while building the app.

## Documents

- `Project-Blueprint.md`: Overall architecture, scope, routes, and quality gates.
- `Phased-Execution-Plan.md`: End-to-end implementation phases with exit criteria.
- `Task-Tracker.md`: Checkbox task list for day-to-day execution.
- `Neo-Brutalist-UI-Guide.md`: Visual system and component style constraints.

## Recommended Execution Flow

1. Read `Project-Blueprint.md` once at kickoff.
2. Follow phases in `Phased-Execution-Plan.md`.
3. Update progress in `Task-Tracker.md` daily.
4. Enforce all UI decisions using `Neo-Brutalist-UI-Guide.md`.

## Change Log Practice

When requirements shift, update these files in this order:

1. `Project-Blueprint.md`
2. `Phased-Execution-Plan.md`
3. `Task-Tracker.md`
4. `Neo-Brutalist-UI-Guide.md`
